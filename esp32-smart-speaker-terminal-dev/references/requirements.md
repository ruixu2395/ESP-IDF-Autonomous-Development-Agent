## 任务 2：完成 APP_Net（网络配网管理）

**输出产物**：
App/APP_Net: app_net.c/app_net.h

**交付要求**：完成源码编写后，自行在 `main.c` 的 `app_main()` 中以非侵入式方式创建 Wi-Fi 配网与连接 Demo。代码需对接中间层 `MID_WiFi` 实现网络栈初始化。自行执行 `idf.py build` 与烧录监控流程，读取串口日志（需使用 `ESP_LOGI` 等规范宏）以确认成功获取 IP 地址且无系统崩溃。完成后暂停操作，请求人类观察串口输出的网络状态，等待人类确认符合预期后完成此任务。

App/APP_BT: app_bt.c/app_bt.h

## 任务 4：完成 APP_IMU（姿态事件处理及底层驱动）

**输出产物**：
driver/driver_imu: driver_imu.c/driver_imu.h （QMI8658C I2C 驱动）
App/APP_IMU: app_imu.c/app_imu.h （计步 / 抬腕事件处理）

**交付要求**：完成源码编写后，在 `main.c` 中创建姿态检测与轮询 Demo。因涉及周期性读取 IMU 传感器数据，**必须**将其封装为 FreeRTOS Task (使用 `xTaskCreate`)，并在 `while(1)` 循环中包含 `vTaskDelay` 以防止触发 Task Watchdog (TWDT) 复位。自行编译烧录并读取串口日志确认任务调度正常。随后请求人类物理晃动或翻转开发板（模拟抬腕动作），观察串口是否有对应的动作识别日志输出，等待现象核对无误后完成此任务。


# 任务开发与要求

每一个大任务可能包含几个子任务，必须严格按照子任务顺序，每次完成后，自己创建demo，自己编译+烧录。等待确认结果，确认反馈通过后才能完成任务，否则需要自行检查代码逻辑。
